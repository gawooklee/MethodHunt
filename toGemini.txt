🛠️ 시나리오 중심 UI 설계 제안 (오버로딩 방지)
가욱 님이 비효율적이라고 느끼지 않도록 UI 레이어를 3단계로 분리하는 걸 추천드려요.

Main Layer (The Console):

퀴즈와 시나리오만 노출합니다.

"이 에러 상황을 해결할 메소드명을 입력하세요."에만 집중하게 만듭니다.

Detail Layer (The Inspector):

정답을 맞히거나 특정 버튼을 눌렀을 때만 extensions(명사/형용사) 정보가 슬라이드 형태로 슥 나타나게 합니다.

마치 코드 에디터에서 함수 위에 마우스를 올리면 라이브러리 설명이 뜨는 것처럼요.

Result Layer (Delta Visualizer):

정답 처리 후, Before가 After로 변하는 Delta 결과값만 짧고 굵게 보여줍니다
